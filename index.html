Herencia

1 / 15

¿Qué es la Herencia?

Un mecanismo que permite a una clase (hija) adquirir los atributos y métodos de otra clase (padre). Modela una relación "es un".

2 / 15

¿Por qué usar Herencia?

Para reutilizar código y crear una jerarquía lógica entre clases. Evita repetir código y organiza mejor el proyecto.

3 / 15

Clase Padre (Superclase)

Es la clase general de la que otras heredan.

class Vehiculo:
    def __init__(self, marca):
        self.marca = marca
    
    def arrancar(self):
        print("El vehículo ha arrancado.")
4 / 15

Clase Hija (Subclase)

Es la clase especializada que hereda de la clase padre.

# Coche "es un" Vehiculo
class Coche(Vehiculo):
    def tocar_claxon(self):
        print("¡Bip, bip!")
5 / 15

Uso de la Herencia

La clase hija tiene acceso a los métodos de la clase padre y a los suyos propios.

mi_coche = Coche("Toyota")
mi_coche.arrancar()    # Método heredado de Vehiculo
mi_coche.tocar_claxon() # Método propio de Coche
6 / 15

Extender el Constructor

A menudo, la clase hija necesita sus propios atributos además de los del padre.

class Coche(Vehiculo):
    def __init__(self, marca, modelo):
        # ¿Cómo le pasamos 'marca' al constructor del padre?
        self.modelo = modelo
7 / 15

La Función super()

Permite llamar a métodos de la clase padre. Se usa para invocar el `__init__` del padre desde el `__init__` del hijo.

class Coche(Vehiculo):
    def __init__(self, marca, modelo):
        super().__init__(marca) # Llama al __init__ de Vehiculo
        self.modelo = modelo
8 / 15

Ejemplo con super()

Ahora el objeto Coche tiene tanto `marca` como `modelo`.

mi_coche = Coche("Ford", "Mustang")
print(mi_coche.marca)  # Ford (del padre)
print(mi_coche.modelo) # Mustang (propio)

Sobrescritura de Métodos

Una clase hija puede redefinir un método de la clase padre para que tenga un comportamiento específico.

class CocheElectrico(Coche):
    def arrancar(self): # Sobrescribimos el método
        print("El coche eléctrico arranca en silencio.")
10 / 15

Herencia Múltiple

Una clase puede heredar de varias clases padre, adquiriendo los métodos de todas ellas.

11 / 15

Ejemplo de Herencia Múltiple

Se listan las clases padre entre paréntesis, separadas por comas.

class Telefono:
    def llamar(self): print("Llamando...")

class Camara:
    def tomar_foto(self): print("Foto tomada.")

class Smartphone(Telefono, Camara):
    pass
12 / 15

Uso de Herencia Múltiple

El objeto `Smartphone` tiene acceso a los métodos de ambas clases padre.

mi_movil = Smartphone()
mi_movil.llamar()      # Heredado de Telefono
mi_movil.tomar_foto()  # Heredado de Camara
13 / 15

Función: isinstance()

Verifica si un objeto es una instancia de una clase o de una de sus superclases.

mi_coche = Coche("Renault")
print(isinstance(mi_coche, Coche))    # True
print(isinstance(mi_coche, Vehiculo)) # True
14 / 15

Función: issubclass()

Verifica si una clase es una subclase de otra.

print(issubclass(Coche, Vehiculo)) # True
print(issubclass(Vehiculo, Coche)) # False
